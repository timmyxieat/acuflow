# 1.09b: Typography Refinements for Appointment Detail

## Context

The appointment detail screen layout is complete, but typography sizes need adjustment per the new typography spec at `docs/specs/typography.md`.

Read the spec first, then apply these specific changes.

---

## Changes Required

### 1. Global Header Back Button (Topbar.tsx)

**Current:** Back arrow icon is 20px (`h-5 w-5`)
**Change to:** 14px (`h-3.5 w-3.5`)

The back button should feel subtle, not prominent. It's a navigation affordance, not a primary action.

```tsx
// In Topbar.tsx, find:
<ArrowLeft className="h-5 w-5" />

// Change to:
<ArrowLeft className="h-3.5 w-3.5" />
```

### 2. Right Panel Time Range (page.tsx - AppointmentHeader)

**Current:** Time range is 18px (`text-lg font-semibold`)
**Change to:** 16px (`text-base font-semibold`)

The time range should be readable but not dominate. It's informational, not a heading.

```tsx
// In AppointmentHeader, find:
<span className="text-lg font-semibold">{timeRange}</span>

// Change to:
<span className="text-base font-semibold">{timeRange}</span>
```

---

## Files to Modify

| File | Change |
|------|--------|
| `src/components/custom/Topbar.tsx` | Back arrow: `h-5 w-5` → `h-3.5 w-3.5` |
| `src/app/appointments/[id]/page.tsx` | Time range: `text-lg` → `text-base` |

---

## Verification

After changes, the visual hierarchy should be:
1. **Patient name** (left panel) - largest, most prominent
2. **Time range** (right panel header) - medium, informational
3. **Back button** - subtle, doesn't compete for attention

The back button arrow should feel like a small navigation hint, not a call-to-action.
